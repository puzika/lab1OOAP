В этой лабораторной работе было продемонстрированно применение паттерна "абстрактная фабрика" на примере веб-приложения для поиска головоломок трех видов (cube, pyraminx, megaminx) по одной из трех категорий (budget, medium, premium). Было реализованно две версии приложения: одна с применением паттерна, другая без. В паттерне используется абстрактный класс Puzzle, который имеет три дочерних класса(Cube, Pyraminx, Megaminx), каждый из которых предстовляет собой один вид головоломок. Также имеется абстрактный класс PuzzleFactory, который также как и класс Puzzle имеет три дочерних класса (BudgetFactory, MediumFactory, PremiumFactory), каждый из которых является фабрикой для создания головоломок по одной из трех категорий. Каждый имеет класс имеет свой ценовой диапозон по которому он отбирает головоломки попадающие в данную категорию. При нажатие на кнопку выбирается одна из трех фабрик, отбираются нужные головоломки и отображаются на страничке. В версии без паттернв изользутся функция displayPuzzles. При нажатие на кнопку, эта функция на вход получает головоломки отобранные по заданному диапозону и отображает из на страничке. Из плюсов абстрактной фабрика можно отметить читабельность кода, что облегчает процесс дебаггинга. Минусом является значительное увеличения количества кода.
